# Seguimiento de Sesiones — Stasisly

## Estado del documento

Este archivo mantiene el registro histórico e informativo de trabajo del
proyecto Stasisly.

No es una fuente normativa ni una autoridad sobre el estado técnico. Una
entrada del tracker no demuestra que código, configuración, infraestructura,
migraciones o funcionalidades existan o funcionen. El estado técnico solo puede
considerarse verificado mediante auditoría y evidencia comprobable.

Debe usarse para registrar sesiones de definición, auditoría, arquitectura,
implementación, QA, release, Customer Success y decisiones relevantes.

No sustituye a los ADRs. Si una sesión produce una decisión arquitectónica,
técnica, de producto, seguridad, pagos, IA, memoria, MCP o release importante,
debe crearse o actualizarse un ADR.

## Reglas de uso

- Registrar solo hechos verificables, decisiones, entregables y pendientes.
- No presentar hipótesis como implementación.
- No ocultar contradicciones.
- No registrar cambios de código si no se han hecho.
- Toda sesión debe tener siguiente paso.
- Toda sesión debe indicar si hubo archivos tocados.
- Toda decisión importante debe vincularse a un ADR o marcarse como pendiente.
- Si Codex/Antigravity interviene, registrar alcance y restricciones.
- Si una sesión termina sin aprobación, marcar el estado como pendiente.
- Registrar una fila en el `Registro de tiempo IA reportado` únicamente cuando
  exista una duración real disponible, enlazándola mediante la misma fecha y
  nombre de sesión de la tabla principal.
- Si Codex u otra herramienta muestra una duración exacta, copiarla
  literalmente, sin convertirla, redondearla ni recalcularla.
- Una duración exacta comunicada explícitamente por el usuario desde la
  interfaz de Codex se considera tiempo reportado válido y debe registrarse
  literalmente.
- Si no existe duración exacta visible o reportada por Codex, no añadir
  sección, fila ni campo de tiempo IA.
- No usar duraciones no exactas ni mediciones manuales como sustituto de una
  duración exacta reportada.
- La duración reportada mide trabajo mostrado por la herramienta IA; no
  demuestra por sí sola calidad, completitud o estado técnico.

## Estados permitidos

Usar uno de estos estados:

- `Planificado`
- `En curso`
- `Completado`
- `Bloqueado`
- `Pendiente de aprobación`
- `Pendiente de auditoría`
- `Existente no auditado`
- `Archivado`

```text
currentAuthState()
getAccessToken()
refreshIfNeeded()
clearSession()
```

Estados implementados: `demo`, `authenticated`, `unauthenticated`, `expired`,

productivo, IA, Stasis Engine, MCP, streaming y adjuntos. No se tocaron
